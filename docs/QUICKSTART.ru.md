# Seedance 2.0 Skill OS — Быстрый старт

> Версия 6.6.0 · От установки до первого «срежиссированного» промпта примерно за 5 минут.
> Полная документация: [README](../README.md).

## Что это

Seedance 2.0 Skill OS — это agent skill, который режиссирует Seedance 2.0 как кинематографист, а не нагромождает прилагательные. Одно правило: **режиссируйте модель, а не управляйте кадром вручную.** Опишите, что делает сцена, — и skill скомпилирует это намерение в готовый к работе промпт.

## 1. Установка (около 5 минут)

Установите этот репозиторий как **одну** корневую skill с именем `seedance-20`; её под-skills и references подгружаются по относительному пути.

**Codex (есть установщик в одну команду):**

```bash
python scripts/install_codex_skill.py --force
```

Копирует репозиторий в `~/.codex/skills/seedance-20` (или `$CODEX_HOME/skills/seedance-20`). Перезапустите Codex и вызовите `$seedance-20`.

**Установка из GitHub (если клиент поддерживает установку по URL репозитория):**

```text
https://github.com/Emily2040/seedance-2.0
```

**Копирование вручную (другие клиенты):** скопируйте папку в каталог skills вашего клиента, сохранив имя `seedance-20`. Типичные пути — проверьте в своём клиенте, это не гарантия поддержки — приведены в [таблице установки в README](../README.md#install): напр. Claude Code `.claude/skills/`, Cursor `.cursor/skills/`, GitHub Copilot `.github/skills/`, Windsurf `.windsurf/skills/`.

> Безопасность прежде всего: устанавливайте только в доверенные agent-клиенты. Перед использованием этой skill в стороннем или незнакомом агенте прочитайте [SECURITY.md](../SECURITY.md).

## 2. Выберите skill под задачу

| У вас есть… | Загрузить сначала |
|---|---|
| смутная идея | `seedance-interview` |
| ясная сцена | `seedance-prompt` |
| история из нескольких клипов | `seedance-sequence` |
| принятый клип для продолжения | `seedance-continuation` |
| плохой или заблокированный результат | `seedance-troubleshoot` |
| персонаж, бренд, знаменитость или реальный человек | `seedance-copyright` |

## 3. Срежиссируйте до того, как писать — четыре вопроса

1. **Что делает сцена?** Поворот, раскрытие, чувство или демонстрация?
2. **Как об этом говорит камера?** Общий план для одиночества, крупный — для лица, наезд — для прозрения.
3. **Что делает свет?** Время суток, жёсткий/мягкий, тёплый/холодный — всё ради намерения.
4. **Что делает звук?** Почти тишина, одна деталь атмосферы или одна реплика.

## 4. Один пример

**Украшательство (слабо):**

```
эпичный кинематографичный кадр: женщина читает письмо, эмоционально, красивый свет, 4K
```

**Режиссура (сильно):**

```
Средний крупный план, на уровне глаз; она опускает письмо, руки замирают, подходит медленный наезд; мягкий свет из окна оставляет лицо простым; почти тишина, один скрип стула.
```

## 5. Два правила, экономящих дубли

- **Сохраняйте теги ссылок точно как написано** — `[Image1]`, `[Video1]`, `[Audio1]`, `@图1`, `@视频1`. Никогда не переводите и не переформатируйте их.
- **Не просите всю историю одной генерацией.** Сгенерируйте Clip 01, посмотрите, чем он *на самом деле* закончился, и напишите Clip 02 от этого реального финала (`seedance-continuation`).

## 6. Безопасность

- **Безопасность контента:** если идея использует защищённого персонажа, знаменитость, бренд, логотип, песню либо лицо или голос реального человека, не прячьте это на другом языке — перепишите в оригинальный, лицензированный или постпродакшн-эквивалент с помощью `seedance-copyright`.
- **Безопасность агента:** этот пакет **не делает сетевых запросов и не содержит телеметрии**; его скрипты детерминированы и работают офлайн. Никогда не вставляйте API-ключи, cookie аккаунта или закрытые материалы в агент, которому не доверяете. См. [SECURITY.md](../SECURITY.md).
